# Band 10 + Android Demo

这个仓库先放一个最小可行 demo 设计骨架，目标是验证下面这条链路：

`手环控制/显示 -> 安卓手机主控 -> 相机/OCR/AI 工作流`

当前内容分成两部分：

- `android-demo/`：安卓端页面、状态流、消息协议 demo
- `vela-demo/`：小米 Vela 手环端页面和消息收发 demo

当前版本有意不接真实蓝牙和真实相机，而是先把这些内容定下来：

- 页面结构
- 状态机
- 手环与手机消息协议
- 文本显示与历史记录逻辑

后续建议按这个顺序接真实能力：

1. 安卓端先接本地假图 + 假 OCR + 假 AI
2. 再接真实 OCR / 真实 AI
3. 再接 Band 10 的 `interconnect`
4. 最后接相机模块
