# 公开拍摄辅助系统

## 一、项目简介

本项目是一个围绕“**相机板拍照 + 安卓手机主控分析 + 小米 Band 10 显示结果**”构建的个人实验性辅助系统。

项目目标不是做一个通用消费级产品，而是把以下链路真正落地并跑通：

1. 相机板执行拍照
2. 图片传输到安卓手机
3. 安卓手机完成图片管理、OCR、AI 分析
4. 安卓手机将实时日志与最终结果同步到手环
5. 手环负责显示过程状态、最终结果和历史记录

整个系统的核心原则是：

- **手机永远是主控**
- **手环只做显示端**
- **相机板只做拍照与传输端**
- **复杂逻辑统一集中在安卓手机端**

---

## 二、项目目标

本项目希望实现一套轻量、清晰、可迭代的多端协同工作流，使系统能够在公开环境下完成：

- 图片采集
- 图片传输
- 本地或远端分析
- 结果显示
- 历史记录管理

项目重点并不是单一某一端，而是三端联动：

- 相机板
- 安卓手机
- 小米 Band 10

当前整体主链路为：

1. 相机板触发拍照
2. 图片回传到手机
3. 手机整理图片队列
4. 手机执行 OCR 或直接调用图像 AI
5. 手机得到分析结果
6. 手机把日志与结果同步到手环
7. 手环显示过程和最终结果

---

## 三、系统组成

### 1. 相机板端

相机板职责尽量保持单一，主要负责：

- 执行拍照
- 缓存当前图片
- 向手机发送图片
- 接收拍照、分析、定时模式等控制指令
- 在定时模式下执行多张拍照与缓存流程

项目当前围绕 ESP32-S3 摄像头板路线推进，并保留：

- 单次拍照
- 自动发图
- Wi‑Fi 主链传图
- BLE 备用控制/传图链路
- SD 卡缓存与批量上传能力

### 2. 安卓手机端

安卓 App 是整个系统的主控核心，负责：

- 管理相机板连接状态
- 接收相机图片
- 维护图片队列
- 管理 AI 接口与分析顺位
- 调用 OCR / AI 能力
- 保存分析历史
- 向手环同步实时日志和最终结果
- 在整个链路中承担“调度中心”的角色

### 3. 手环端

手环当前基于小米 Band 10 / Vela 应用方案实现，职责包括：

- 显示实时日志
- 显示分析结果
- 查看历史记录
- 查看历史详情
- 接收过程通知和震动提示

手环不承担 OCR、AI、图片处理等重逻辑，只承担轻量展示职责。

## 四、当前功能方向

本项目围绕以下几个方向持续迭代：

### 1. 单次拍照分析

支持完成：

- 单次拍照
- 图片传输到手机
- 手机分析
- 手环显示结果

### 2. 多图队列分析

支持：

- 多张图片加入手机图片队列
- 后续统一分析
- 分析成功后再根据规则清空旧队列

### 3. 定时拍照模式

支持：

- 定时拍照请求
- 按固定时间间隔拍摄多张图片
- 每张图片缓存到 SD 卡
- 全部拍完后统一上传到手机
- 上传完成后自动开始分析

### 4. 手环实时状态显示

手环端可显示：

- 相机状态
- 上传状态
- 分析状态
- 当前顺位 / API
- 最终结果
- 历史记录与详情

### 5. 自定义 AI 路线

项目当前以“自定义 AI 接口”为主，支持：

- 自定义 URL
- 自定义 API Key
- 自定义模型
- 分析顺位切换
- 失败后按顺位继续尝试

---

## 五、项目特点

本项目并不是单独某一个页面或某一个小工具，而是一个真实的三端联动实验工程。

主要特点包括：

- 手机端作为主控，职责清晰
- 手环端轻量化，只显示不做重逻辑
- 相机板端尽量简单，降低负载与发热
- 支持日志、结果、历史、图片队列、定时模式等完整流程
- 保留继续迭代和扩展的空间

此外，项目在实现上优先考虑：

- 分阶段联调
- 主链路优先
- 先稳定，再扩展
- 尽量减少四端同时大改带来的联调风险
